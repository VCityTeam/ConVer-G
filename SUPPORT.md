# Getting help with ConVer-G

Thanks for using ConVer-G! If you need help, here is where to look.

## Documentation

- The [README](Readme.md) covers installation, the data model, and how to run
  each component.
- Each module has its own `README.md` (for example
  [quads-delta/README.md](quads-delta/README.md)) with module-specific details.
- A fully reproducible end-to-end example is available in the
  [UD-Demo-VCity-Knowledge_Evolution reproducibility guide](https://github.com/VCityTeam/UD-Demo-VCity-Knowledge_Evolution/blob/JOSS-ConVer-G/Reproducibility.md).

## Asking a question

If the documentation does not answer your question, please open an issue on the
[issue tracker](https://github.com/VCityTeam/ConVer-G/issues) using the
**Question / support** template. Describe what you are trying to do and what you
have already tried — this helps us help you faster.

## Reporting a bug

Please open a [bug report](https://github.com/VCityTeam/ConVer-G/issues/new/choose)
and include, where possible:

- the version (release tag or commit) you are running,
- the component involved (`quads-loader`, `quads-query`, `quads-visualizer`, …),
- the steps to reproduce, the expected behaviour, and what actually happened,
- relevant logs or error messages.

## Requesting a feature

Feature ideas are welcome — open a
[feature request](https://github.com/VCityTeam/ConVer-G/issues/new/choose)
describing the problem you want to solve and, if you have one, a proposed
solution.

## Contributing

If you would like to contribute a fix or improvement yourself, see
[CONTRIBUTING.md](CONTRIBUTING.md).
